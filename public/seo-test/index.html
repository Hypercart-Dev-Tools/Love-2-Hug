    How to Use

    Five steps. Works with Lovable, Cursor, Copilot, and more.

    Drop it in. Point your AI at it. Audit and quickly fix what comes back.

    Step 01

    Upload AGENTS.md

    Drop the file into your Lovable chat before your first draft. It contains everything — including embedded templates for DASHBOARD.md and QUICKHELP.md.

    Step 02

    Extract & Follow

    Instruct Lovable to follow AGENTS.md and extract the helper files from the appendices.

    Follow AGENTS.md and extract DASHBOARD.md and QUICKHELP.md from the appendices (see §0.0).
    Step 03

    Audit First Draft

    Request a compliance audit against the matrix in DASHBOARD.md. Catch structural issues before they compound.

    Audit the code against the Compliance Matrix in DASHBOARD.md and report any violations.
    Step 04

    Each Build Cycle

    Reset the DASHBOARD.md checkboxes and update header dates. Run the loop again.

    Step 05

    When Bugs Pile Up

    Re-audit compliance before adding features. A broken contract upstream is often the root cause — faster than chasing individual bugs.

    Audit codebase against DASHBOARD.md Compliance Matrix. Flag violations. Fix: security → data → UX.
